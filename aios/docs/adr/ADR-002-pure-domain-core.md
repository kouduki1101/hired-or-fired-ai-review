# ADR-002: ドメインコアは外部I/Oゼロの純関数とする(依存はNumPyのみ)

- 状態: 採用 (2026-07-05)

## 文脈
指標演算(EMA/散逸度/適合度)と制御判断(Rehatch選定/ダイナミクス)は特許の中核であり、
監査時に「同一入力から同一判断」を再現(リプレイ)できる必要がある。

## 決定
- `packages/core` はHTTP/DB/ファイルI/Oを持たない。乱数はシードを引数で受ける
- ポリシー判断は入力(SlotView等の値オブジェクト)→出力(選定結果+理由)の純関数
- 依存規約はimport-linter(CI)で強制する

## 帰結
- (+) ユニットテスト容易(カバレッジ90%目標)、判断の決定的リプレイが可能
- (+) 散逸度アルゴリズム等のStrategy差し替えが安全
- (−) アプリ層(worker/orchestrator)にI/O接着コードが必要 → 各workerは「取得→core呼出→保存」の薄い形に統一
