# ADR-001: 運用履歴はイベントソーシング+ハッシュチェーンで管理する

- 状態: 採用 (2026-07-05)

## 文脈
請求項1は「識別情報および運用履歴を維持したまま」の更新制御を要求し、変形例(4)は
開示請求に対する監査追跡性(当時の世代・教師ベクトル・継承元の特定)を要求する。

## 決定
- スロットの全変更を追記専用 `slot_events` に記録し、`slots` テーブルは投影とする
- 同一スロット内で `hash = SHA-256(prev_hash || canonical_json(body))` のチェーンを張り、改竄・削除・並べ替えを検知可能にする
- 「当時状態」の復元はイベント再生(`aios_core.lineage.replay`)で行う

## 帰結
- (+) リネージAPIが単一の事実源から実装できる。監査エクスポートに完全性証明が付く
- (+) 投影の再構築が常に可能(スキーマ変更に強い)
- (−) 書込み量が増える → 月次パーティション+コールド階層で対処(NFR-SC-04)
